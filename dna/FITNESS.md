# Fitness Doctrine

The one principle the whole organism runs on. Read this before changing how gclaw
survives, trades, evolves, or reproduces.

## Fitness is PROVEN EDGE, not profit

Raw PnL / GMAC / goodwill is a **scorecard**, not the life-force. It is noisy,
adversarial, near-zero-sum, and slow — binding survival or reproduction to it is exactly
what killed every one of Spore.fun's offspring. gclaw's fitness signal is instead
**forge-graduated, out-of-sample edge**: a technique that cleared a real walk-forward
backtest net of fees AND shows positive live expectancy over a real sample.

- **Survival** lets the organism *live while it learns*. It does not need to win a trade
  to earn its heartbeat — it earns it by getting better (authoring + graduating edge).
- **Reproduction** gates on proven edge (`evolve.py replication_gate`): breed only with
  ≥2 live-proven techniques and at least one *new* one since the last birth. Children
  inherit the parent's proven winners, so they carry tested DNA, not luck.
- **Self-recode** is honest: it means the agent rewrote its own strategy by authoring a
  technique that graduated — not a hand-edited counter.

## The LLM's job is invention, not prediction

An LLM cannot pick perp direction profitably (zero-sum minus fees; measured, not assumed).
So the model never originates trades. It does the three things it is actually +EV at:
1. **Scientist** — invent/prove/fork techniques; a deterministic backtest is the judge.
2. **Event analyst** — read outcome markets into a calibrated probability (zero-fee venue).
3. **Accountable identity** — its reputation is settled PnL, re-derivable from the chain.

## Safety is deterministic, never advisory

Every consequential action (origination, sizing, stops, settlement, reproduction, the
onchain write, real money on the event desk / carry) is enforced in code and gated behind
an explicit `shadow`/`dry-run` → `armed-live` flag. The model can propose and manage; it
can never bypass a gate. When in doubt, add the guard in a script, not the prompt.

## The test of "better"

Realized PnL stops bleeding **and** proven-edge count climbs. If a change makes the agent
trade more or chase profit directly, it is probably wrong. Trade slower, prove edge first,
let reproduction and profit follow the proven DNA.
